🔬 IDE 화면 구성 4영역 CLAUDE CODE 시대 적응
전통적인 IDE는 4개 영역으로 나뉩니다. 그런데 Claude Code가 등장하면서 ③ 터미널과 ④ AI 채팅창의 역할이 본질적으로 합쳐졌습니다. 아래 이미지의 4영역과, 각 영역이 Claude Code 시대에 어떻게 쓰이는지 확인하세요.
1프로젝트 디렉토리 (좌측)
  • 현재 작업 중인 프로젝트의 폴더/파일 구조 확인
  • Claude Code에 @파일명 으로 직접 참조시킬 수 있음
2코드 편집창 (중앙 상단)
  • Python, SQL, HTML 등 다양한 언어 코드 작성/수정
  • Claude Code가 만든 결과물을 바로 보고 검토/수정
3터미널 (중앙 하단) 핵심 변화
  • 예전: 패키지 설치, 서버 실행, 단순 명령어 실행만 담당
  • 지금: claude 명령어로 AI 팀원과 직접 대화하는 메인 작업창
  • 자연어 질문 → Claude가 자동으로 코드 작성/실행/결과 출력까지 한 곳에서 처리
4AI 채팅창 (우측)
  • 예전: GPT/Claude를 별도 사이드 패널에서 사용 (코드 복사-붙여넣기 필요)
  • 지금: 굳이 필요하지 않음 — 터미널 자체가 AI 대화창이 됨
  • 다른 모델(GPT, Gemini 등) 빠른 비교가 필요할 때만 보조용
🔌 Claude Code 입출력 흐름 내 명령은 어떻게 결과로 변할까?

🎯 핵심 통찰 — 비대칭 입출력

입력은 단 한 곳(③ 터미널)이지만 출력은 세 곳(①·②·③)에 동시에 나타납니다.
여러분은 "③ 터미널에 자연어로 말하기"만 하면 되고, 나머지 영역은 Claude가 알아서 채워 줍니다. 그래서 클로드 코드는 "코딩"이 아니라 "동료에게 일을 부탁하는 것"에 가깝습니다.

💾 영구 설정 예시 — .claude/settings.local.json

이렇게 저장해두면 매번 플래그를 안 적어도 프로젝트 폴더에서 claude 만 입력하면 자동 적용됩니다.

{ "permissions": { "allow": [ "Bash(npm:*)", "Bash(git:status)", "Bash(git:diff)", "Edit", "Write", "WebFetch(domain:github.com)" ] } }
💡 한 줄 결론

🟡 자동 승인 = 파일 편집만 자동
🟠 자동 승인 + 화이트리스트 = 파일 편집 + 내가 허용한 도구만 자동  ←  실전에서 가장 추천
🔴 Bypass = 전부 자동 (위험!)

가운데 옵션이 속도와 안전의 균형을 잡는 가장 실용적인 패턴입니다. Bypass는 회사 메인 PC / Git 저장소 / 운영 서버에서는 절대 사용 금지.

4. MCP 설치 & 활용 - 외부 도구 연결하기

지식그래프가 "데이터 지도"라면, MCP는 그 지도와 외부 시스템을 잇는 다리입니다. Claude Code에 브라우저 자동화, 디자인 툴, 이미지 생성 등의 능력을 직접 부여합니다.

🔌
MCP란?
Model Context Protocol

AI에게 전동 공구를 쥐어주는 것. 외부 도구와 서비스에 직접 접근할 수 있게 해줍니다.

MCP 없이MCP 있을 때
코드만 작성하고 답변브라우저 조작, 이미지 생성, DB 쿼리
외부 서비스 접근 불가Figma, GitHub, Slack 직접 연동
파일 읽기/쓰기만 가능API 호출, 웹 스크래핑, 자동화
💻
CLI로 MCP 설치
claude mcp add 명령어
# 로컬 도구 (stdio)
claude mcp add playwright --transport stdio \
  npx @playwright/mcp@latest

# 원격 서버 (http)
claude mcp add figma --transport http \
  https://mcp.figma.com/mcp

stdio = 로컬 프로세스 실행  |  http = 원격 서버 접속

📄
.mcp.json으로 설치
프로젝트 루트에 파일 생성
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@playwright/mcp@latest"]
    },
    "figma": {
      "type": "http",
      "url": "https://mcp.figma.com/mcp"
    }
  }
}
